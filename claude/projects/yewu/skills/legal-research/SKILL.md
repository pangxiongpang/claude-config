# 法律检索

## 功能

使用威科先行数据库检索法律法规和裁判文书，获取原文和案例详情。

## 与「法条核实」的分工

| 技能 | 职责 | 数据源 |
|------|------|--------|
| **法律检索**（本 skill） | 查找法条原文、搜索类似案例 | 威科先行 |
| **法条核实** | 核实在用法条是否现行有效 | 国家法律法规数据库 |

> 检索到法条后，如需核实其是否现行有效，调用 **法条核实** skill。

## 流程

1. **确认案情**：先问清楚，不自行推测
2. **确认关键词**：提炼3-5个，展示给用户确认
3. **执行检索**：
   ```bash
   cd d:/AI/1.业务/.claude/skills/legal-research && python legal_search.py "关键词" "裁判文书" --detail
   ```
   **自动保存**：输出目录 `C:\Users\11828\OneDrive\obsidian\胖胖熊\案件\raw\案件\案例\`
4. **输出报告**：保存到 Obsidian（指定案件写 `案件/案件名/类似判例检索报告.md`，未指定写 `检索报告/`）

## 参数

- 第一个参数：搜索关键词
- 第二个参数：`法律法规` 或 `裁判文书`（默认裁判文书）
- `--detail`：抓取详情页内容（**必须同时传入关键词搜索，searchId只在本次会话有效，不可复用过期结果URL**）

## 注意事项

- **searchId单次会话有效**：详情URL带 `searchId` 参数，过期后无法复用。需要重新跑 `--detail`。
- **获取全文**：脚本已配置为获取判例完整全文，不做截取。
- **不要写调试脚本**：页面结构问题直接用 `legal_search.py` 跑一遍确认，它已经完成了登录→搜索→导航→提取全流程。单次排查写一行 `page.screenshot()` 即可。**不要写多个独立调试脚本层层排查**。

## 硬性约束

- 必须先确认案情，不自行推测
- 必须确认关键词，展示给用户后再检索
- 案号不可省略
- 禁用威科先行AI功能
