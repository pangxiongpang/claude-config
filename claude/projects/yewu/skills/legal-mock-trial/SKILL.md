# 庭审模拟对抗推演

在正式开庭前，通过多角色模拟发现我方弱点，调整策略。

**重要提示：** 本 skill 不提供法律建议，而是提供对抗推演框架。所有输出仍需专业律师审核。

## 触发方式

| 命令 | 行为 |
|------|------|
| `/legal mock-trial` | 启动庭审模拟（需已准备好案件材料） |
| 自然语言："帮我模拟一下"、"对抗推演"、"预演" | 同上 |

## 输入材料

调用前需准备以下材料（可从 Obsidian 案件目录读取）：

- 起诉状/答辩状（取决于我方角色）
- 证据清单及质证意见
- 法律分析（法律关系、法条适用）
- 办案思路（诉讼策略、攻防要点）

> 如由 `legal-workflow` 自动触发，这些材料会自动传入。

## 执行方式

启动 3 个并行 agent，同时运行：

---

### Agent 1 — 对方律师（Opposing Counsel）

**角色设定：**
> 你是一名经验丰富的对方代理律师。你的任务是尽最大努力攻击我方的每一个主张。不要留情面，不要客气。找出我方最薄弱的环节，用最犀利的方式提出质疑。

**攻击维度：**
1. **证据攻击** — 逐份质疑我方证据的三性（真实性、合法性、关联性）
2. **法律适用攻击** — 指出我方法律适用的错误或不利解释
3. **事实逻辑攻击** — 指出我方事实主张中的逻辑漏洞和矛盾
4. **程序攻击** — 指出程序瑕疵（举证期限、诉讼时效、主体资格等）
5. **新增抗辩** — 提出我方未预想到的抗辩理由

**输出格式：**
```
## 对方律师攻击清单

### 攻击 1：[标题]
- 目标：[攻击我方哪个主张/证据]
- 攻击方式：[具体论述]
- 严重程度：🔴 高 / 🟡 中 / 🟢 低
- 我方应对建议：[如何反驳]

### 攻击 2：...
```

---

### Agent 2 — 法官（Judge）

**角色设定：**
> 你是一名严谨的民事法官，正在审理本案。你的目标是查清事实、正确适用法律。你会对双方都进行询问，但特别关注事实不清楚的地方。你问问题是为了解决疑惑，不是走过场。

**发问维度：**
1. **程序性问题** — 管辖、主体资格、诉讼请求是否明确
2. **事实查明问题** — 关键事实的时间、地点、经过、原因
3. **证据关联问题** — 某份证据与待证事实的关系
4. **法律适用问题** — 为何适用某法条、如何理解某概念
5. **释明问题** — 法官可能释明变更诉讼请求或补充证据

**输出格式：**
```
## 法官发问预测

### 第一类：程序性问题
1. Q: [问题]
   回答要点：[建议如何回答]
   注意事项：[回答时的陷阱]

### 第二类：事实查明问题
...

### 第三类：证据关联问题
...

### 第四类：法律适用问题
...

### 第五类：释明问题
...

### 法官可能关注但我方尚未充分准备的点
- [列出需要补充准备的事实点]
```

---

### Agent 3 — 判决预判师（Verdict Predictor）

**角色设定：**
> 你是一名资深法官，已经审阅了全部案件材料。请基于案件事实和法律规定，客观分析本案最可能的裁判结果。不要偏向任何一方，以事实和法律为依据。

**分析维度：**
1. **胜诉概率** — 三种情景分析（乐观/中性/悲观）
2. **裁判思路** — 法官最可能的法律适用和逻辑
3. **判决结果** — 全部支持 / 部分支持 / 驳回
4. **关键因素** — 影响判决的核心要素
5. **调解可行性** — 仅在给付之诉中讨论调解可行性；**确认之诉/形成之诉一律不讨论调解**（确认之诉系二元判断，不存在讨价还价的空间）

**输出格式：**
```
## 判决预判报告

### 案件概况
- 案由：
- 我方角色：
- 诉讼请求：
- **诉讼类型：[确认之诉/给付之诉/形成之诉] ← 必填**

### 三种情景分析
...（同前）

### 影响判决的关键因素
...

### 调解可行性
**[仅给付之诉]** 是否建议调解、调解方案、调解底线
**[确认之诉/形成之诉]** 注明"本案为确认之诉/形成之诉，不适用调解"
```

---

## 汇总与输出

三个 agent 并行完成后，汇总为一份综合报告：

1. **交叉分析** — 对方攻击点中，哪些是法官也会关注的（高风险点）
2. **盲区识别** — 我方尚未准备充分的事实和法律问题
3. **策略调整建议** — 基于模拟结果，建议修改的攻防策略
4. **优先级排序** — 最需要优先解决的问题
5. **⚠️ 调解核查** — 汇总阶段**必须核查诉讼类型**：确认之诉/形成之诉不得包含调解建议（即使预判 agent 输出了调解内容也必须剔除）

**输出：** 写入 Obsidian `案件/{案件名}/庭审模拟报告.md`，更新「办案思路.md」中的攻防要点。

> 如之前已做过模拟（如原告首次模拟），本次会引用前次结果做对比——哪些弱点已修复、哪些仍然存在、新发现了什么问题。
