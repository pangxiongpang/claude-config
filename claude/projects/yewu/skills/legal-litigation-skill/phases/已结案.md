# 已结案

## 目标

复盘 → 提炼实务要点 → 更新知识库。

## 工作流

1. **案件复盘**：
   - 做得好的（胜诉原因、关键决策、有效策略）
   - 做得不好的（失误、延误、判断错误）
   - 核心教训（可跨案复用的知识点）
   - 警示清单（同类案件需要特别注意的事项）
2. **创建反思页面**（在 `案件/wiki/反思/` 下，以核心教训命名）
3. **提炼实务要点**（在 `案件/wiki/专题/实务要点/` 下）
4. **更新案件状态**：
   - `案件.md` 状态改为"已结案"
   - `index.md` 已更新
   - `log.md` 已更新
   - `todo.md` 已移除本案待办

## 产出

- [ ] 反思页面
- [ ] 实务要点（如适用）
- [ ] 案件已结案标记
- [ ] 知识库已更新
