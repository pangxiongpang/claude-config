# 立案前

本阶段含两个子阶段：**案件受理** → **立案准备**，均在立案前完成。

---

## 子阶段 A：案件受理

### 目标

建立案件主页面 → 创建关联知识 → 启动第一轮任务追踪。

### 工作流

1. **创建案件主页**：按标准结构创建 `案件.md`
2. **当事人分析**：
   - 对方当事人信息
   - 对方可能的诉讼策略预判
3. **案件事实梳理**：时间线、关键证据、争议焦点
4. **检索知识库**：生成 `关联知识.md`（见 CLAUDE.md §5.8）
5. **创建进展摘要**：按 `assets/summary-template.md` 格式
6. **更新案件状态**：同步 `todo.md` → `index.md` → `log.md`

### 产出

- [ ] `案件.md` — 案件主页面（含基本信息、当前任务、当事人、案件事实）
- [ ] `关联知识.md` — 知识库检索结果
- [ ] `进展摘要.md`
- [ ] `todo.md` / `index.md` / `log.md` 已更新

### 注意事项

- 当事人信息务必**区分客观事实与当事人单方陈述**。对于当事人主张但无书面证据支撑的内容，在案件事实中标注"据当事人陈述"。
- 案件事实模块**必须引用具体的证据**，不能单纯描述当事人陈述。如果当事人说的内容没有证据佐证，则注明"无书面证据支撑，待核实"。
- 起诉状、证据清单等详细文书由子阶段 B 处理，本子阶段只建框架。

---

## 子阶段 B：立案准备（核心压力阶段）

### 目标

起草起诉状/答辩状 → 证据清单 → 庭审模拟 → 定稿。

### 工作流

1. **读案件主页**：确认当事人、案由、事实、争议焦点、当前任务
2. **金额计算**：列明细，包含本金、利率、起止日期、计算公式
3. **诉讼请求分解**：
   - 逐项检查每一条诉请：有对应事实描述 + 有对应法条依据
   - 诉请之间无逻辑矛盾
   - 每一项都能对应到证据
4. **证据清单**：逐份标注证据名称、来源、证明目的、三性评价
5. **庭审模拟（委派 `legal-mock-trial`）**：
   - 将案件事实、争议焦点、对方信息打包委派
   - 接收模拟报告，将攻击点反馈到诉请/证据调整
6. **根据模拟结果调整**诉请/补强证据
7. **委派专业 skill 执行子任务**：

   | 委派对象 | 任务 | 输入 |
   |----------|------|------|
   | `legal-templates` | 起草起诉状/答辩状 | 案件事实、诉讼请求、法条依据 |
   | `law-validation` | 核实每条引用的法条现行有效 | 法条名称+条号列表 |
   | `legal-research` | 类案检索 | 案由+争议焦点+关键词 |

### 产出

- [ ] 起诉状/答辩状
- [ ] 证据清单（逐份标注三性）
- [ ] 金额计算明细表
- [ ] 庭审模拟报告
- [ ] `进展摘要.md` / `todo.md` 已更新

### 注意事项

- 列法条时注意**区分实体法（如民法典）与程序法（如民诉法）**。不同的法层级不同，律师需要同时关注二者，否则容易遗漏程序性权利。
- 金额是本阶段最高频出问题的项，每一次交互都要检查金额明细。
- 尽量别让用户一次性给太多信息，分步引导，一次一个焦点。每完成一小步就确认。
- 起诉状定稿后，提示用户检查：法院管辖权是否正确？诉讼时效是否已过？
- 委派 skill 后，必须验证产出质量：法条是否正确、逻辑是否自洽、格式是否符合要求。
