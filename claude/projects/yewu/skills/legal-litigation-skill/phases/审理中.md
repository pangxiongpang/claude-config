# 审理中

本阶段含两个子阶段：**举证与庭前** → **开庭**，均在审理中完成。

---

## 子阶段 A：举证与庭前

### 目标

证据深度分析 → 不利事实识别 → 质证准备 → 庭审提纲。

### 工作流

1. **读进展摘要** + 案件主页（确认阶段已推进）
2. **Claim Chart 分析（委派 `legal-evidence`）**：
   - 提供案件背景 + 请求权基础 + 现有证据列表
   - 要求输出每项请求权要件→对应证据→标记证据缺口
3. **不利事实分析（委派 `legal-evidence`）**：
   - 输出 8 类不利事实识别 + 严重程度评估 + 应对策略
4. **五维风险评估（委派 `legal-evidence`）**：
   - 输出证据/法律/对方/执行/时间 逐维评分
5. **关联证据文件委派**（如适用）：
   - `legal-review` — 合同类证据深度分析
   - `legal-compare` — 多份合同版本比对
6. **质证意见准备**：基于委派产出，整理逐份证据的三性意见
7. **庭审提纲**：法官可能追问的问题清单 + 回答要点

### 产出

- [ ] Claim Chart
- [ ] 不利事实分析报告
- [ ] 五维风险评估
- [ ] 质证意见
- [ ] 庭审提纲

### 注意事项

- 委派上游 skill 时，必须提供三要素：**案件背景、具体任务、输出格式要求**。缺少任何一项都可能导致产出不符合预期。
- 质证意见建议区分"真实性异议"、"合法性异议"、"关联性异议"三种类型。不能笼统说"三性均不认可"。
- 庭审提纲要站在法官视角：法官拿到案卷后会关注哪些问题？

---

## 子阶段 B：开庭

### 目标

庭前检查 → 代理词定稿 → 开庭。

### 庭前检查清单

- [ ] 传票所载时间、地点、法庭确认无误
- [ ] 代理词已打印（含法庭陈述、质证发言、辩论提纲、最后陈述）
- [ ] 法律依据汇编已整理打印
- [ ] 证据材料已按顺序整理（原件 1 套 + 复印件 N 套）
- [ ] 质证意见已打印
- [ ] 庭审提纲已打印
- [ ] 授权委托手续已携带
- [ ] 律师事务所函已开具
- [ ] 所需法条/判例已打印备用

### 注意事项

- 此阶段相对轻量，重点是检查清单执行
- 如需委派 skill，仅 `legal-templates`（代理词模板）和 `legal-mock-trial`（庭前模拟）
