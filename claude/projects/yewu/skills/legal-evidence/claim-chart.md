# Claim Chart（请求权-证据映射表）

## 什么是 Claim Chart

一张表格，左边列请求权的每个构成要件，右边列对应的证据。用于检查"我的证据能不能撑起我的请求权"。

## 使用步骤

### 第一步：确定请求权基础

明确我方主张的请求权是什么，例如：
- 买卖合同：请求支付货款
- 民间借贷：请求偿还借款
- 侵权纠纷：请求损害赔偿
- 劳动争议：请求支付经济补偿

### 第二步：拆解构成要件

将请求权拆解为法律构成要件。以"请求支付货款"为例：

| 要件编号 | 构成要件 | 法律依据 |
|---------|---------|---------|
| ① | 合同成立 | 民法典第490条 |
| ② | 卖方已交货 | 民法典第604条 |
| ③ | 货款金额 | 合同约定 |
| ④ | 付款期限已届满 | 合同约定 |
| ⑤ | 催告事实 | 民法典第563条 |
| ⑥ | 未付款事实 | 举证责任 |

### 第三步：逐要件映射证据

| 要件 | 对应证据 | 证据强度 | 缺口 |
|------|---------|---------|------|
| ①合同成立 | 购销合同原件 | ✅ 强 | 无 |
| ②卖方已交货 | 送货单+签收单 | ✅ 强 | 无 |
| ③货款金额 | 对账单 | ✅ 强 | 无 |
| ④付款期限已届满 | 合同约定"货到30日付款" | ✅ 强 | 无 |
| ⑤催告事实 | 微信催款记录 | ⚠️ 中 | 需补充书面催告函 |
| ⑥未付款事实 | 银行流水（无对应付款记录） | ✅ 强 | 无 |

### 第四步：缺口检测

**输出缺口清单**：

```
## 证据缺口

### 缺口1：催告事实
- 要件：证明已向对方催告
- 现有证据：微信催款记录（⚠️ 中等强度）
- 缺口：无书面催告函
- 补强建议：
  1. 发送书面催告函（EMS留存）
  2. 微信记录做公证固定
  3. 收集对方回复记录
```

## 证据强度判定

| 强度 | 标准 | 示例 |
|------|------|------|
| ✅ 强 | 原件/公证/对方确认 | 合同原件、公证文书、对方盖章确认函 |
| ⚠️ 中 | 复印件/间接证据/单方制作 | 复印件、聊天记录截图、单方记账 |
| ❌ 弱 | 证据瑕疵/可能不被采信 | 无法核实的录音、缺失关键信息 |

## 适用场景

- **复杂商事案件**：请求权基础多，需要逐一检查
- **知产案件**：侵权构成要件复杂
- **庭前自检**：开庭前检查证据链完整性
- **代理词论证**：按要件逐一论述，逻辑清晰
